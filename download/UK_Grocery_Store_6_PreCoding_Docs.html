Product Requirements Document

App Name and One-Line Idea

UK Local Grocery Store

A hyperlocal grocery delivery platform that enables UK customers to browse, order, and receive groceries delivered to their door in as fast as 15 minutes, with full UK legal compliance including VAT handling, allergen display, Challenge 25 age verification, and GDPR data protection.

Target Users and Problem Statement

User TypeDescriptionKey Pain Points
CustomersUK residents ordering groceries online for home deliverySlow delivery, inaccurate ETAs, hidden fees, missing allergen info, no age verification transparency
Store OwnersIndependent grocery shop operators managing online ordersNo integrated system for orders, inventory, staff, and compliance; manual processes lead to errors
ManagersDay-to-day operations staff overseeing picking and deliveryOrder tracking across chat/calls, no real-time visibility, inventory mismatches
PickersStaff who collect and pack items from shelvesNo structured packing workflow, quality assurance gaps, no QA checklist
RidersDelivery personnel fulfilling last-mile deliveryNo map integration, no ETA control, no age verification at door, poor communication with customer

User Stories

  • As a customer, I want to enter my postcode and instantly know if delivery is available, so that I don't waste time browsing unavailable services.
  • As a customer, I want to see unit pricing (e.g., 1.40 GBP/100g) on every product, so that I can compare value fairly.
  • As a customer, I want allergens highlighted in bold on product pages, so that I can quickly identify safety risks.
  • As a customer, I want a live countdown timer when my order is out for delivery, so that I know exactly when to expect it.
  • As a rider, I want to manually set my delivery ETA based on local knowledge, so that customers get accurate time estimates.
  • As a rider, I want a "Cannot Verify" button for Challenge 25 items, so that age-restricted items are handled legally.
  • As a manager, I want a Kanban board that flashes when new orders arrive, so that I can assign pickers and riders immediately.
  • As an owner, I want VAT collected per month broken down by rate (0%, 5%, 20%) with CSV export, so that I can file MTD returns.
  • As a customer, I want to delete my account and have my data anonymised, so that I can exercise my GDPR right to erasure.
  • As a picker, I want a QA checklist on the packing screen, so that I can verify item quality before dispatch.

Customer Journey — End to End

1
Postcode Gate — Customer enters or GPS-detects postcode. Validated against delivery_zones via Postcodes.io. Invalid → "Sorry, we don't deliver to your area" + email signup.
2
Browse Homepage — Banner carousel, category sliders, product cards with unit pricing, discount badges, 18+ badges.
3
Search or Navigate — Live predictive search or category sidebar navigation. Products filtered by sub-category instantly.
4
View Product — Full detail: images, allergens (bold), ingredients, use-by date, unit price, Challenge 25 warning. Add to Cart.
5
Review Cart — Slide-out sidebar. Quantity controls, bill summary, promo code, delivery fee, age-restricted item warning.
6
Checkout — Address confirm, order summary, VAT breakdown, cancellation rights, 18+ confirmation checkbox. Proceed to Pay.
7
Payment — Stripe 3D Secure modal. Server creates PaymentIntent, verifies total from DB (not client). Webhook confirms.
8
Order Confirmed — Email receipt via Resend, order appears in Kanban "New" column (flashing + sound alert).
9
Packing — Picker accepts order, follows QA checklist, marks as Packed.
10
Dispatch — Manager assigns Rider. Rider reviews address + map, sets manual ETA (15/30/45/60 min), taps Start Delivery. Customer receives SMS + push with countdown.
11
Live Tracking — Customer sees countdown timer, status timeline, rider info. Rider can adjust ETA (+10 min, max 3 times).
12
Delivery — Challenge 25 check at door. Rider marks Delivered. Confirmation email + SMS to customer. Feedback prompt.

Login / Signup Flow

Customer: Enter email → magic link sent → click link → authenticated (7-day session). Guest checkout available with email-only. Post-purchase: prompt to set password for full account.

Staff: Enter email + password → optional TOTP 2FA → authenticated (30-min idle session). All sessions via HTTP-only cookies.

Mobile App: First launch → location permission → GPS postcode detection → biometric login (Face ID / fingerprint) → fallback to magic link.

Payment Flow

Customer clicks "Proceed to Pay" → Server creates Stripe PaymentIntent with amount calculated from DB (not client) → Stripe 3D Secure modal appears → Customer authenticates → Stripe sends webhook to /api/payments/webhook → Server verifies webhook signature → Marks order "Paid" → Decrements inventory → Sends email receipt → Kanban "New" column flashes.

Failure path: Payment fails → Order marked "Abandoned" → Inventory restored → Customer receives retry email → Cart preserved 15 minutes.

UI/UX Design Brief

Design Style and Mood

Style: Clean, modern, trust-building grocery retail. Minimalist with bold accent colours for calls-to-action. Mobile-first with thumb-friendly interaction zones.

Mood: Fast, reliable, fresh. The design should communicate speed (delivery timer, quick-add buttons) and trust (compliance badges, secure payment indicators, clear pricing).

Philosophy: Search-first, not browse-first. Most grocery shoppers know what they want — the search bar is the hero. Categories and sliders support discovery for undecided shoppers.

Mobile / Desktop Behavior

Mobile-first: All designs start from 375px and scale up. Thumb zones prioritised — CTAs in bottom 60% of screen. Minimum tap target 44x44px (WCAG). Swipe gestures for cart and categories.

Desktop: Expanded navigation, multi-column grids, sidebar cart becomes inline. Admin dashboard fully featured on desktop with data tables and charts.

Rider mobile web: Optimised for one-handed use on phone browsers. Large touch targets (48x48px min), high contrast, tel: links for calling, embedded map view.

Overall UX Principles

  • Speed perceived: Delivery timer, skeleton loading (200ms), optimistic UI updates for cart
  • Trust signals: Stripe badge, VAT breakdown, allergen emphasis, 30-day guarantee statement, ICO registration
  • Compliance visible: Unit pricing on every card, Challenge 25 badges, cookie consent, cancellation rights at checkout
  • Glanceable information: Countdown timer, order status timeline, low stock badges — no scrolling for critical info
  • Forgiving design: Cart preserved on payment failure, 15-min retry window, guest checkout, ETA adjustment grace

Data Ownership Rules

  • Customers can only access their own orders, profile, and addresses
  • Pickers can only see orders in "New" and "Packing" status
  • Riders can only see orders assigned to them in "Out for Delivery"
  • Managers can see all orders and modify products, categories, banners, promo codes, and staff
  • Owners have unrestricted access to all data including finance and audit logs
  • All data modifications are logged in audit_log with user_id, action, timestamp, and before/after values

Authentication / Session Handling

Session Storage: Vercel KV (Redis) via HTTP-only cookies. Session ID is a cryptographically random token.

Customer Sessions: 7-day expiry, no idle timeout. Extend on each request.

Staff Sessions: 30-minute idle timeout. 2FA optional (TOTP). Force re-auth on role elevation.

Guest Checkout: Cart stored in Vercel KV with session ID. No user account required. Post-purchase prompt to create account.

Token Storage: Stripe tokens never stored in our database. PaymentIntent IDs stored for reconciliation only.
